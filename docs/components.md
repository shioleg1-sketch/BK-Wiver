# Компоненты BK-Wiver

## Desktop

Платформы:

- `Windows`
- `macOS`

Назначение:

- единое приложение, которое можно установить на компьютер пользователя;
- после установки приложение может работать в двух ролях:
  - как узел, к которому подключаются;
  - как оператор, который подключается к другим узлам.

Основные функции:

- логин на сервере;
- публикация текущего устройства на сервере;
- список доступных устройств;
- создание исходящей remote-session;
- прием входящей remote-session;
- просмотр и управление удаленным рабочим столом;
- файловый обмен;
- диагностика разрешений и состояния подключения.

Внутренние модули:

- `control-plane` для регистрации устройства и heartbeat;
- `session-manager` для входящих и исходящих сессий;
- `screen-capture`;
- `input-injection`;
- `file-transfer`;
- `permissions-diagnostics`;
- `desktop-ui`.

## Server

Платформы:

- `Ubuntu Server 24.04 x86_64`

Назначение:

- обрабатывает запросы от desktop-приложений;
- хранит каталог устройств и пользователей;
- аутентифицирует запросы;
- координирует создание сессий;
- ведет аудит событий;
- отдает web-интерфейс управления.

Минимальный состав:

- `HTTP API`;
- `WebSocket signaling`;
- `web admin UI`;
- каталог устройств;
- каталог пользователей;
- аудит;
- выдача временных токенов;
- интеграция со `STUN/TURN`.

## Внутренние роли внутри Server

На старте сервер может быть одним приложением. Логически внутри него нужно разделить:

- `auth`;
- `device registry`;
- `session coordinator`;
- `audit service`;
- `web admin`.

Физически выносить их в отдельные сервисы на первом этапе не нужно.
