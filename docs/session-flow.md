# Поток сессии BK-Wiver

## 1. Подготовка

На каждом компьютере установлен `Desktop`.

Перед началом remote-session:

1. пользователь логинится в `Desktop`;
2. `Desktop` регистрирует локальное устройство на `Server`;
3. `Desktop` открывает signaling-соединение с `Server`;
4. `Desktop` периодически шлет heartbeat.

## 2. Исходящее подключение

Пусть `Desktop A` хочет подключиться к `Desktop B`.

Последовательность:

1. `Desktop A` запрашивает список устройств.
2. Пользователь выбирает `Desktop B`.
3. `Desktop A` вызывает `POST /api/v1/sessions`.
4. `Server` создает `sessionId` и short-lived `sessionToken`.
5. `Server` отправляет `session.request` на `Desktop B`.

## 3. Принятие подключения

Если на `Desktop B` разрешены входящие подключения:

1. пользователь подтверждает подключение или срабатывает unattended policy;
2. `Desktop B` отправляет `session.accepted`;
3. оба клиента получают параметры сессии;
4. начинается обмен `offer / answer / ice_candidate`.

Если подключение запрещено:

1. `Desktop B` отправляет `session.rejected`;
2. `Server` закрывает сессию;
3. `Desktop A` показывает причину отказа.

## 4. Установление WebRTC

После подтверждения:

1. `Desktop A` создает `offer`;
2. `Desktop B` отвечает `answer`;
3. обе стороны обмениваются `ICE candidate`;
4. WebRTC выбирает прямой маршрут или идет через `TURN`.

## 5. Каналы данных и медиа

После установления peer connection:

- video stream: удаленный экран;
- data channel `input`: мышь и клавиатура;
- data channel `control`: служебные команды;
- data channel `files`: операции передачи файлов;
- опционально `clipboard`: буфер обмена.

## 6. Завершение сессии

Сессия завершается если:

- пользователь нажал disconnect;
- истек `sessionToken`;
- одна из сторон потеряла связь;
- удаленная сторона отозвала разрешение;
- сервер принудительно завершил сессию.

После завершения:

1. стороны отправляют `session.closed`;
2. сервер пишет audit event;
3. `Desktop` возвращается к экрану списка устройств.

## 7. Что важно для MVP

На первом этапе лучше не усложнять поток:

- одна сессия между двумя устройствами;
- один оператор на одну удаленную машину;
- без аудио;
- без групповых подключений;
- без постоянной истории чатов и комментариев.
