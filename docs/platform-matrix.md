# Платформенная матрица BK-Wiver

## Windows

### Захват экрана

- основной кандидат: `DXGI Desktop Duplication`;
- запасной путь для отдельных сценариев: `Windows Graphics Capture`.

### Ввод

- инъекция мыши и клавиатуры через WinAPI;
- отдельно проверить сценарии с `UAC secure desktop`.

### Запуск

- системный сервис;
- отдельная логика для взаимодействия с пользовательской сессией.

### Риски

- переключение пользователей;
- блокировка экрана;
- работа в средах RDP;
- ограничения secure desktop.

## macOS

### Захват экрана

- современные версии: `ScreenCaptureKit`;
- запасной вариант для старых систем: legacy API, если потребуется.

### Ввод

- потребуются `Accessibility` permissions;
- для части операций могут понадобиться дополнительные entitlement и подписание.

### Запуск

- `launchd`;
- нужно разделять фоновый сервис и UI/permission helper, если это понадобится.

### Риски

- `TCC` и выдача прав;
- подпись приложения и notarization;
- отличия поведения между локальной сессией и headless-сценариями.

## Linux X11

### Захват экрана

- `X11/XShm/XCB` в зависимости от выбранной библиотеки.

### Ввод

- `XTest` или эквивалентные механизмы.

### Запуск

- systemd user/service в зависимости от модели установки.

### Риски

- большой разброс дистрибутивов;
- несовпадение пакетов и библиотек;
- несколько display server конфигураций.

## Linux Wayland

### Захват экрана

- `PipeWire` + `xdg-desktop-portal`.

### Ввод

- значительно сложнее, чем на X11;
- часть окружений ограничивает или запрещает синтетический ввод.

### Запуск

- нужна проверка совместимости с GNOME, KDE и другими compositors.

### Риски

- непредсказуемые ограничения compositors;
- разные portal-реализации;
- unattended access может быть проблемным без отдельных обходных стратегий.

## Вывод для MVP

Практичный порядок платформ:

1. `Windows` как первая полнофункциональная платформа.
2. `macOS` как вторая по приоритету.

Для раннего MVP разумно явно ограничить поддержку так:

- полнофункциональный `Desktop`: `Windows`;
- `Desktop` с оговорками по системным разрешениям: `macOS`.
