# Оптимизация качества изображения и FPS для BK-Wiver

## 🎯 Цель
Достижение качества изображения и 30 FPS для системы удаленного доступа.

## 📊 Текущее состояние

### Кодирование и декодирование
- Поддерживаемые кодеки: H.264, VP8
- Hardware acceleration: NVENC, QSV, AMF, VideoToolbox
- Текущий FPS: ~20-25 (недостаточно)
- Текущее качество: HD/SD (можно улучшить)

## 🔧 Оптимизации для качества и FPS

### 1. Оптимизация кодеков

#### H.264 (рекомендуемый выбор)
```rust
// Оптимальные настройки для H.264
const H264_CRF: u8 = 25;        // Качество (20-30)
const H264_BITRATE: u64 = 5000000;  // 5 Mbps для 1080p
const H264_PRESSET: &str = "slow";      // Preset для качества
```

#### VP8
```rust
// Настройки для VP8
const VP8_BITRATE: u64 = 3000000;   // 3 Mbps
const VP8_FPS: u32 = 30;             // Целевой FPS
const VP8_KEYFRAME_INTERVAL: u32 = 50;
```

### 2. Оптимизация буферизации

#### Увеличение буфера кадров
```rust
// Хранить последние N кадров для интерполяции
const FRAME_BUFFER_SIZE: usize = 5;  // 5 кадров в буфере

// Предпросмотр предыдущих кадров
fn predict_next_frame(&self, frames: &[Frame]) -> Frame {
    // Интерполяция между предыдущими кадрами
    interpolate_frames(frames, FRAME_BUFFER_SIZE)
}
```

### 3. Оптимизация захвата экрана

#### Адаптивное разрешение
```rust
// Динамическое разрешение по качеству сети
fn get_optimal_resolution(network_quality: u8) -> (u32, u32) {
    match network_quality {
        90..=100 => (1920, 1080),  // Отлично
        70..=89  => (1280, 720),   // Хорошо
        50..=69  => (854, 480),    // Удовлетворительно
        _        => (640, 480),    // Плохо
    }
}
```

### 4. Оптимизация GPU ускорения

#### NVENC настройки
